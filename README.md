# Игра: Путь Судьбы
## Описание проекта
Консольная RPG-игра, разработанная на Python, с тремя сюжетными ветками, системой артефактов и пошаговой боевой системой. Игра реализует все требования задания: три сюжетные линии, систему сохранения прогресса, копилку артефактов и интерактивные выборы, влияющие на развитие сюжета.
## Особенности игры
1. Три сюжетные ветки

Путь Воина - фокус на физической силе и чести

Путь Мудреца - акцент на магии и знаниях

Путь Странника - свобода и приключения


2. Система сохранения прогресса
   
Автоматическое сохранение персонажа в JSON-формате

Пароли хранятся в открытом виде (по требованию задания)

Возможность загрузки предыдущих сохранений

3. Копилка артефактов

Коллекционирование уникальных артефактов

Возврат артефактов в копилку при отказе от сохранения

Автоматическая генерация новых артефактов при сборе всех существующих


4. Боевая система

Пошаговые бои на основе классов из исходного кода

Система характеристик (сила, ловкость, интеллект)

Боссы с двумя фазами боя


5. Система выбора

Ветвящиеся сюжетные линии

Каждый выбор влияет на развитие истории

История всех выборов сохраняется

## Структура проекта 

game.py - Основной файл игры

player.py - Класс игрока и система сохранения

kopilka.py - Система артефактов

users.json - База данных пользователей (автосоздание)

artifacts.json - База данных артефактов (автосоздание)

saves/ - Папка с сохранениями (автосоздание)

README.md - Документация

## Установка и запуск

### Требования

Python 3.6 или выше

Только стандартные библиотеки Python (без внешних зависимостей)

## Управление игрой

Игра полностью управляется через консольный интерфейс. Все действия выполняются путем выбора цифровых опций из меню.

## Игровой процесс

### Создание персонажа

1. Регистрация - создание учетной записи с логином и паролем

2. Выбор пути - выбор одной из трех сюжетных веток

3. Начальные бонусы - каждый путь дает уникальные бонусы к характеристикам

### Прохождение сюжета

Каждая сюжетная ветка предлагает три варианта развития событий. В каждом варианте:

1. Исследование - погружение в историю выбранного пути

2. Бой - пошаговое сражение с противником

3. Награда - получение артефактов, золота или опыта

### Боевая система

**Ход игрока:**

Атаковать (базовый урон)

Использовать умение (усиленная атака)

**Ход противника:**

Автоматическая атака

Боссы имеют две фазы боя

### Система артефактов
Получение - артефакты награда за победы в битвах

Хранение - все артефакты хранятся в инвентаре игрока

Возврат - при отказе от сохранения все артефакты возвращаются в копилку

Генерация - новые артефакты генерируются при сборе всех существующих

## Система сохранения
### Автоматическое сохранение
Данные сохраняются в папку saves/

Каждый пользователь имеет свой файл сохранения (username.json)

Сохраняются все характеристики, артефакты и история выборов

### Особенности сохранения

1. Пароли в открытом виде - реализовано по требованию задания

2. Опциональное сохранение - игрок может отказаться от сохранения

3. Последствия отказа - все артефакты возвращаются в копилку, прогресс теряется

# Архитектура кода
## Классы игровой системы
### Human (базовый класс)
Базовый класс для всех существ в бою

Содержит основные атрибуты: здоровье, мана, характеристики

Реализует базовые методы атаки и получения урона

### Warrior (наследник Human)
Специализируется на физическом уроне

Атака: базовая сила + случайный бонус (1-3)

### Mage (наследник Human)
Специализируется на магическом уроне

Атака: интеллект × 2 (требует ману)

### Boss (наследник Human)
Особый противник с двумя фазами боя

Вторая фаза (ярость): усиленная атака (+5 урона)

### Player (класс игрока)
Управляет всеми данными игрока

Реализует систему сохранения/загрузки

Управляет инвентарем артефактов

### Kopilka (копилка артефактов)
Управляет пулом артефактов

Реализует генерацию новых артефактов

Обрабатывает возврат артефактов

### Game (основной класс игры)
Координирует весь игровой процесс

Управляет меню и сюжетными ветками

Связывает все компоненты системы

## Расширение игры
### Добавление новых сюжетных веток
1. Создать новый метод в классе Game (например, story_new_branch())

2. Добавить логику ветвления в метод choose_story_branch()

3. Реализовать уникальные выборы и награды

### Добавление новых классов персонажей
1. Создать новый класс-наследник Human

2. Реализовать уникальный метод attack()

3. Интегрировать в систему создания персонажа

### Добавление новых артефактов
1. Добавить названия в список artifacts_pool в kopilka.py

2. Реализовать уникальные эффекты (при необходимости)

3. Обновить логику генерации артефактов
